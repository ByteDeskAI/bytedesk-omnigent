# Codegen spike verdict — ServerStreamEvent 44-variant union (BDP-2444, ADR-0152, TS half)

**Question:** can the 44-variant `ServerStreamEvent` discriminated union — and the
runtime needed to turn an SSE frame into a narrowed type — be 100% generated by
`openapi-typescript` alone, or is a custom generator required?

## Verdict: `openapi-typescript` emits the union TYPE but not the runtime → a custom generator is used.

`openapi-typescript@7` was run against the pinned `src/contract/openapi.json`. The
omnigent schema *does* carry a `ServerStreamEvent` `oneOf` + `discriminator`, so
the tool did model it — but only as a **bare structural union of the 44 variant
component types**:

```ts
ServerStreamEvent:
  components["schemas"]["SessionStatusEvent"]
  | components["schemas"]["SessionUsageEvent"]
  | … (44 members) …
```

What it does **not** give you — and what a usable SSE SDK needs — is the runtime:

1. **No `type`-keyed runtime dispatcher.** It's a compile-time union; there is no
   `parse(json) -> variant` function. Going from a decoded SSE frame to a narrowed
   `ServerStreamEvent` would be hand-written regardless. (TypeScript *narrows* the
   union on `switch (ev.type)`, but only once you already hold a value typed as the
   union — the cast from raw JSON is the part that has to be authored.)
2. **No `UnknownEvent` tolerance / known-discriminator set.** A frame whose `type`
   the pinned schema doesn't know (version skew) has nowhere to land — there is no
   escape hatch and no set to test membership against, so a newer server event
   would silently fall outside the union.
3. **Generates the whole HTTP surface, not just the union.** Its job is the broad
   `paths` / `components` / `operations` DTOs; the event union is one type buried in
   `components`, with the same name as a 1500-line file of unrelated DTOs — not a
   focused, importable union with its own runtime helpers.

This is the exact shape of the C# spike verdict (NSwag flattened the union into a
single bag class with no STJ polymorphism — see the C# `codegen/SPIKE.md`),
reproduced for TypeScript: a stock generator emits a *type*, not the *machinery*.

## What we build instead (still 100% generated, no hand edits)

- **`openapi-typescript`** owns all HTTP DTOs (`src/generated/openapi-types.ts`):
  `paths`, `components`, `operations`. The SDK re-exports `components` / `paths` /
  `operations` for callers that want the broader surface.
- **`codegen/generate-events.mjs`** (this folder) reads
  `src/contract/server-stream-event.schema.json` and emits
  `src/generated/server-stream-events.ts`: the 44 variant interfaces (each pinning
  its `type` literal), the 12 shared sub-objects they reference
  (`ResponseObject`, `Usage`, …), the `ServerStreamEvent` discriminated union, an
  `UnknownEvent` escape hatch, the `SERVER_STREAM_EVENT_TYPES` known-set, and
  `parseServerStreamEvent(json)` which dispatches on `type` and **never throws** on
  an unknown frame. This generator owns ONLY the event union + its sub-objects;
  it is self-contained from the event schema (no cross-reference to the
  openapi-typescript `components` names).

The two generators have non-overlapping outputs, so there is no name clash and no
double-definition of the variant types.

## Reproducibility

`npm run regenerate` reproduces both `src/generated/*.ts` byte-for-byte from the
pinned contract. The event generator emits `$defs` and properties in schema order
(the schema document is canonicalized with `sort_keys=True` upstream) with no
timestamps, so a clean run yields an empty `git diff` under `src/generated/`.
Never hand-edit generated output.

## Drift gate

`src/contract/contract.lock` pins the sha256 of both snapshot files + the omnigent
git sha they were taken from. `contract-drift.test.ts` fails if a contract file
changes without re-locking, and `every-wire-type.test.ts` fails if the schema's
discriminator set and the generated union diverge (the "every wire `type` has a
variant, never silently drop" guarantee). Both hashes are byte-identical to the C#
SDK's `contract.lock` — the two SDKs are pinned to the same omnigent snapshot
(`962a88fb`).
